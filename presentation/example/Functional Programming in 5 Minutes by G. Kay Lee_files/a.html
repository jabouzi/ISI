            Brian McKenna Bartosz Zawada 4 years ago

            Referential transparency allows equational reasoning. In short, whenever you see function application you can replace it with that function's code but with arguments replaced with what you passed in (look at beta reduction from lambda calculus for an example). You can now reason about these programs as just function calls. That's all your program becomes.

            When you perform side-effects, you don't have referential transparency. You can't reason about your program as just function application, you have to think about it about state transitions (e.g. o.x was 1 and calling function f makes o.x equal to 2). It's much harder to prove that your code will do what you intended. You'll certainly have more bugs.

                ezrakilty Brian McKenna 4 years ago

                That explains the benefits of equational reasoning, but not of functional programming in JavaScript. Equational reasoning doesn't apply in JavaScript because it's not pure: it's hard to know whether a given function has side effects or not.

                A functional style of programming within an imperative language is still valuable for several reasons. For one, you can at least use equational "intuition" in some places (although it's a dangerous way to reason). But more often, functional techniques just give you flexibility to do things in new ways, which might save you time, depending on what path you take in your hacking process.

                (I'd argue that the technique demonstrated in this presentation is just a programming technique, which happens to come from the community of functional programming. It is especially valuable in concert with other functional-programming techniques, but it is a useful tool even without those.)

                Currying functions and otherwise using function transformers may allow you to solve some programming problem that would otherwise be more laborious to solve. For example, you may find yourself wanting a whole family of callbacks that only differ by some parameter (say an animation speed). Partial application gives you an easy way to create those.
